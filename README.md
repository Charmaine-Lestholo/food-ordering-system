\# \*\*PROJECT INVESTIGATION:\*\*



\##### 1. What is Spring Boot?

Spring Boot is an opinionated framework built on top of the Spring ecosystem that makes it fast and straightforward to build production-ready applications. it is mostly used for building REST APIs, Web applications and Microservices. Spring boots key features are, autoconfiguration, embedded servers, production-ready tools and microservices support.



\##### 2.What is Maven?

Maven is a build automation tool developed in Java, mainly used to manage and streamline the build process of Java-based applications. It simplifies tasks like compilation, testing, and packaging through a centralized configuration. Mavens offers features like dependency management, standard project structure, build lifecycle, plugins, POM file and more.



\##### 3. What is the purpose of pom.xml?

The pom.xml file is the main configuration file used by Maven. It contains project information, dependencies, plugins, and build settings.



\##### 4. What is the purpose of application.properties?

The application.properties file stores application configuration settings such as database connection details, server ports, and other application-specific properties.



\##### 5. What does SpringBootApplication do?

SpringBootApplication is an annotation that combines configuration, component scanning, and autoconfiguration. It serves as the main entry point of a Spring Boot application.



\##### 6. Why do developers use dependency management tools such as Maven?

Dependency management tools help developers automatically download, manage, and update external libraries required by a project, reducing manual work and compatibility issues.



\##### 7. What is a REST API?

A REST API is a web service that allows applications to communicate over HTTP using methods such as GET, POST, PUT, and DELETE.



\##### 8. What is JSON?

JSON (JavaScript Object Notation) is a lightweight data format used to exchange information between systems in a structured and readable way.



\##### 9. What is Dependency Injection?



&#x20;  Dependency Injection is a design pattern where objects receive their dependencies from an external source rather than creating them directly. This improves flexibility, maintainability, and testability.



\# Package Structure

\##### 1.controller

Contains classes that handle incoming HTTP requests and return responses.



\##### 2.service

Contains business logic and processing rules for the application.



\##### 3.repository

Contains interfaces responsible for database operations and data access.



\##### 4.entity

Contains classes that represent database tables.



\##### 5.dto

Contains Data Transfer Objects used to transfer data between layers of the application.



\##### 6.config

Contains configuration classes and application settings.



\##### 7.exception

Contains custom exception classes and error-handling logic.
